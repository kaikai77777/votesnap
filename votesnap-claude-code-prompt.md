# VoteSnap.online 改版指令（Claude Code）

> 給 Claude Code：請依以下 Phase 順序執行，**每完成一個 Phase 先停下來輸出變更摘要與測試檢查清單，等我確認後再進下一個 Phase**。先掃描現有代碼結構（前端 framework、後端 API、資料庫 schema、認證流程）並向我報告，再開始動工。

## 0. 專案背景與改版原則

VoteSnap.online 是一個讓使用者匿名發起「二選一決策投票」的 PWA。目前已上線，現有設計語言要**完整保留**：

- 深色主題（純黑底 #000）
- 紫 → 粉 → 橘漸層作為主色（`#A855F7` → `#EC4899` → `#F97316` 區間）
- 圓角膠囊按鈕、卡片式排版
- `votesnap` 漸層 logo
- 既有頁面：首頁、發問、熱榜、我的問題、設定、定價、社群規範

### 改版鐵則
1. **不重寫整體架構**，在現有代碼上做精準修改
2. **不更動視覺風格**（顏色、字體、圓角、漸層方向都保持一致）
3. 每個改動都要附上：影響的檔案、變更摘要、需要測試的場景
4. 資料庫 schema 變動需先列出 migration 計畫等我同意
5. 任何破壞性改動（刪欄位、改 API 回傳格式）先警告

## Phase 0：緊急修復（信任與轉換殺手）

### 0.1 投票免登入（最重要）

**現況問題：** 朋友收到分享連結點進來，被強制 Google 登入才能投票，估計 70%+ 訪客在這一步流失。

**改動：**
- 投票（點選 Yes/No、提交一票）**完全不需要登入**
- 防重複機制改為：`device fingerprint`（FingerprintJS 或自實作）+ `IP` + `localStorage UUID` 三層判定
- 同一指紋對同一題只能投一票，前端顯示「你已經投過了」
- **發問仍需要登入**（保留現有 Google OAuth 流程）
- 如果使用者已登入，投票記錄關聯到 user_id；未登入則記錄 anonymous_id

**資料庫：**
- `votes` 表新增 `anonymous_id` 欄位（nullable），與 `user_id` 至少有一個非空
- 加入唯一索引：`(question_id, user_id)` 與 `(question_id, anonymous_id)`

**驗收：**
- 隱身視窗開啟分享連結 → 直接看到題目 → 點 Yes → 看到結果
- 同一隱身視窗再投同題 → 顯示「已投過」
- 換瀏覽器投同題 → 可以投

---

### 0.2 解鎖「投票紀錄」（移除 Pro 限制）

**現況問題：** 「我的問題」頁的「投票紀錄」分頁鎖在 Pro 後面（顯示鎖頭與升級按鈕）。這是用戶自己的資料，鎖住等於懲罰用戶。

**改動：**
- 投票紀錄分頁無條件對所有登入用戶開放
- 移除 Pro 鎖頭 UI 與「升級 Pro」CTA
- Pro 在這裡的差異化改為「**進階分析**」：投票傾向圖表、與全站趨勢的相似度、AI 個性洞察（先放占位，下一階段做）

**檔案：** `MyQuestions` 或類似 page，找到 `isPro` gating 邏輯移除

---

### 0.3 隱藏首頁的 0 數據卡

**現況問題：** 首頁顯示「0 今日發問 / 0 累計投票 / ~15 min 平均結果」，跟上方「直播中 — 現在有人在投票」自相矛盾，是嚴重信任殺手。

**改動：**
- 三個數據卡的顯示邏輯：
  - **累計投票 < 1000：** 整區隱藏
  - **累計投票 ≥ 1000：** 只顯示「累計票數」單一卡片，文案改為「累計 12,847 次選擇障礙被治好」
  - **今日發問**只有當 ≥ 20 才顯示
- 「~15 min 平均結果」如為靜態文案則直接刪除；如為計算值，當樣本 < 50 題時隱藏

**驗收：** 訪客在站內活動量低時看不到任何「0」

---

### 0.4 新用戶登入後保證有題可投

**現況問題：** 新用戶第一次登入後立刻看到「全部投完了！」（綠色打勾 + 暫時沒有新問題了），首因效應災難。

**改動：**
- 投票 feed 演算法調整：
  - 優先取「未投過 + 進行中」的題
  - 如不足 20 題，回補「已結束但票數 ≥ 50」的熱門題（標註為「熱門回顧」）
  - 永遠不顯示空狀態，除非全站總題數 < 5
- 「全部投完了」空狀態只在使用者確實投完所有可投題目時出現

---

### 0.5 熱榜門檻

**現況問題：** 熱榜第一名 3 票、第二名 2 票、第三名 1 票，看起來像鬼站。

**改動：**
- 熱榜只列出**票數 ≥ 10** 的題目
- 「今日」分頁：票數 ≥ 5（門檻較低，因為時間窗短）
- 「本週」分頁：票數 ≥ 10
- 「全部」分頁：票數 ≥ 30
- 不夠門檻時，熱榜顯示空狀態：「還在累積熱門投票中，先去看看其他題吧」+ 跳轉到投票 feed 的按鈕

## Phase 1：產品結構修正

### 1.1 投票時效調整

**現況問題：** 現在預設值偏短（10 分鐘），朋友訊息看到時容易已過期。

**改動：**
- 發問頁時效選項改為：
  - 免費用戶：**15 分鐘 ⚡ / 1 小時 / 1 天（預設）**
  - Pro 用戶額外解鎖：**3 天 / 7 天 / 30 天**
- **預設值改為「1 天」**，避免新手誤選太短時效造成沒人投票的挫敗感
- 「15 分鐘」選項 UI 加閃電 icon ⚡ 標示為「快閃」用途
- Pro 選項 UI 加 Pro 徽章（鎖頭 icon），點擊未訂閱用戶時跳轉定價頁

---

### 1.2 開放 2–4 個選項

**現況問題：** 只支援 Yes/No 兩選項，「三家餐廳選哪間」「三件衣服哪件好看」這類最常見的選擇障礙場景無法使用。

**改動：**
- 發問頁的「選項」區塊改為動態列表：
  - 預設 2 個（保留 Yes/No 為快速範本按鈕）
  - 「+ 新增選項」按鈕，最多 4 個
  - 每個選項可附一張圖（圖片題型）
- 投票結果頁的長條圖支援 2–4 條
- 分享圖（share image）排版相應調整：2 條時維持現狀；3–4 條時自動縮排
- 資料庫：`options` 表已存在的話保留；如硬編碼為 yes/no 兩欄則需 migration

---

### 1.3 Emoji 反應

**現況問題：** 純投票太冷，沒有「被看到」的感覺。

**改動：**
- 投票結果卡片下方新增 emoji 反應列：🤣 😱 ❤️ 🤔 👀
- 任何人（含未登入）可點，每個 emoji 顯示計數
- 防重複：同 anonymous_id/user_id 同題目同 emoji 只能 +1，再點取消
- 不需要留言區（先做這個就好）

---

### 1.4 Web Push 推播

**現況問題：** 沒有任何回訪 hook，使用者投完就消失。

**改動：**
- PWA 已有 Service Worker 的話直接接 Web Push API；沒有的話建立
- 推播時機：
  1. 自己發的題收到第 5 / 10 / 50 / 100 票
  2. 自己發的題即將結束（剩 1 小時）
  3. 自己發的題結果出爐
  4. （未來）每日「靈魂拷問」推播
- 設定頁新增「通知設定」區塊，可細項開關以上四種
- 第一次投完票後彈出推播授權詢問（非首次互動就詢問，授權率太低）

---

### 1.5 「我的問題」改列表為預設

**現況問題：** 預設日曆視圖過度設計，新用戶只發過幾題還要點日期找。

**改動：**
- 「我的問題」分頁預設**列表視圖**（按時間倒序）
- 日曆視圖改為次要選項，加切換按鈕（icon 切換：列表 / 日曆）
- 列表項目：題目、選項、目前票數、狀態（進行中 / 已結束）、發布時間相對化（「2 小時前」）

## Phase 2：差異化與增長

### 2.1 發問頁模板與 AI 提示

**改動：**
- 發問頁標題下方新增「**今天熱門問法**」橫向滑動區，顯示 5 個範本（從近期高互動題萃取）：
  - 「要不要傳訊息給她？」
  - 「這件衣服值得買嗎？」
  - 「消夜吃 A 還是 B？」
  - 點擊範本自動填入問題框
- 「不知道怎麼問？」按鈕呼叫 LLM API（建議用 Claude Haiku，成本低速度快）：
  - 輸入：使用者一句話描述情境
  - 輸出：3 個改寫過的二選一/多選一問題
- 自動分類：使用者輸入問題後，呼叫 LLM 判斷分類（愛情/職場/購物/生活/旅遊/美食/健康/科技/運動/其他），自動勾選並讓使用者可改

---

### 2.2 低票數分享圖優化

**現況問題：** 分享圖（圖 15）在 0 票時顯示「Yes 0% / No 0%」看起來像 bug。

**改動：**
- 分享圖根據票數動態切換版型：
  - **< 5 票：** 隱藏百分比，顯示「快幫我選 →」+ 呼籲動畫元素
  - **5–50 票：** 顯示百分比 + 票數
  - **> 50 票：** 顯示百分比 + 票數 + 「🔥 熱門投票」標籤

---

### 2.3 動態 OG Image

**現況問題：** 連結貼到 LINE/Threads 的預覽卡片不知道長怎樣，但若是純 logo 預覽則點擊率慘。

**改動：**
- 建立 `/api/og?id=<question_id>` endpoint，動態生成 1200×630 的 OG image：
  - 包含問題本文
  - 包含當前票數比例（小型長條）
  - 底部 votesnap 漸層 logo + 「點擊投票 →」
- 每個 question detail page 的 `<meta property="og:image">` 指向這個 endpoint
- Twitter Card 用 `summary_large_image`
- 用 `@vercel/og` 或 `satori` 實作（如為 Next.js）

---

### 2.4 熱榜分眾

**改動：**
- 熱榜頁頂部新增分類 tab：「**全部** / 我的興趣 / 愛情 / 職場 / 購物 / 美食 / ...」
- 「我的興趣」根據使用者個人設定（圖 13 的興趣標籤）過濾
- 未登入或未設定興趣時，「我的興趣」tab 隱藏

---

### 2.5 Pro 訂閱重新定位

**現況問題：** 現在的 Pro 福利（無限發問、優先曝光、投票紀錄、60/120 分時效）多半是「閹割免費版來換的」，而不是真價值。

**改動 — Pro 福利重組為：**
1. **投票者人口統計**：發問者可看「投 Yes 的人 vs 投 No 的人」的性別、年齡分布（基於用戶設定資料，匿名聚合，最少 10 票才顯示避免反推個資）
2. **AI 結果洞察**：每題結束後一段 AI 生成的解讀（「你的圈子比全站平均更傾向 Yes，這通常代表⋯⋯」）
3. **私密投票**：產生只有特定連結能投的題目，不上熱榜，適合內部決策
4. **客製分享圖**：選背景、字體、加自己的浮水印
5. **長期投票時效**（3 天 / 7 天 / 30 天，免費版最長 1 天）
6. **進階篩選**：在投票紀錄看自己的選擇傾向圖表
7. **無廣告**（如未來放廣告）

**移除的 Pro 福利：**
- 「投票紀錄」（改為全部開放，已在 0.2 處理）
- 「無限發問」改為「每日 30 題」（從 5 題提升），維持稀缺感避免濫發
- 「優先曝光」這個詞太空，移除或改為「Pro 題目顯示徽章」
